What is HTTP?

  • HTTP is a stateless communication protocol.
  • Servers do not keep information about clients in–between requests.
  • HTTP provides support for other functionalities such as:
    • Cache control
    • Content Media Type (MIME) Specification
    • Language and Character set specification
    • Content/ Transfer codings
    • Content negotiation
    • Client-Server Protocol negotiation
    • Persistent Connections
    • Request Pipelining/Multiplexing
    • Authentication/Authorization

HTTP RESOURCE ADDRESSING

HTTP Resources are identified using URLs (RFC 3986) or more specifically, HTTP URLs

  • Uniform Resource Locators(URL)

    • Uniform Resource Locators (URL) is a type of URI. URL is a website address that identifies the location of a website or a file. The URL contains the protocol to be use followed by the ip address and domain name.

  • Scheme

    • http or https

  • Authority

    • User information or authentication credentials(deprecated)

  • Domain Name

    • Resolve to an IP Address using DNS of the server where the resource resides or will be created

  • Path to Resource

    • Resolve relative to the document root on the server. It may also refer to a static or dynamic resource

  • Query

    • Typically provided as key-value pairs, with ampersand(&)separators between key/value pairs.

  • Fragment Identifier

    • Pound Sign (#)

HTTP MESSAGE: REQUEST AND RESPONSE

HTTP message is a blocks of data that is sent by client to the server and vice versa. There are two kinds of HTTP message namely request and response message.

Response

It is the reply of the server to the client. The client will send a request message with the following format. The server will now then receive the message and interpret it and respond to it with the same message format.

Request

  • Messages that is sent by client to the server.
  • (CRLF –term line consisting of 3 space separated values)
  • Method
  • Request Target
  • Protocol Version

HTTP MESSAGE STRUCTURE

  • Start-line - A single line that text that describes the request and determine if the request is successful or not.
  • Headers - An area in which a message request that contains informaation about the body.
  • Body - a.k.a PAYLOAD, Contains a block of data usually a text, html file, image, etc. either request from a client or response from a server.

STANDARD METHODS

METHOD PROPERTIES
Get - query safe / idempotent / cacheable
Head - no body, metadata only safe / idempotent / cacheable
Post - part of the body cacheable
Put - requests that the enclosed entity be stored under supplied Request-URI idempotent
Delete - not supported by servers idempotent
Options - it allows the client to determine the options and requirements associated with the resource safe / idempotent
Trace - used to invoke a remote loop-back of the request message safe / idempotent
Connect - establish a tunnel/connection

Safe Methods

  • Read - only

Idempotent Methods

  • Effects of multiple request is the same as a single request

Cacheable Methods

  • Allowed to store

HTTP STATUS CODES

  • 100 - Continue
  • 101 - Switching Protocols

2xx: Success

  • 200 - OK
  • 201 - Created
  • 202 - Accepted
  • 203 - Non-authoritative Information
  • 204 - No Conten
  • 205 - Reset Content
  • 206 - Partial Content

3xx: Redirection

  • 300 - Multiple Choices
  • 301 - Moved Permanently
  • 302 - Found
  • 303 - See Other
  • 304 - Not Modified
  • 305 - Use Proxy
  • 306 - Unused
  • 307 - Temporary Redirect

4xx: Client Error

  • 402 - Payment Required
